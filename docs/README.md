# Docs

| Path                                            | Audience              | What's here                                          |
|-------------------------------------------------|-----------------------|------------------------------------------------------|
| [`developer/SETUP.md`](developer/SETUP.md)       | First-timers          | Install everything from scratch (WSL, uv, VSCode)    |
| [`developer/DEVELOPMENT.md`](developer/DEVELOPMENT.md) | Day-to-day contributors | Daily workflow: lint, test, CV, notebooks, debugging |
| [`developer/ARCHITECTURE.md`](developer/ARCHITECTURE.md) | New maintainers       | Why this stack, package layout, data flow           |
| [`developer/AGENTS.md`](developer/AGENTS.md)     | Programmers using AI agents | Per-harness setup (Claude Code, Codex, Gemini, Aider, …), agent workflow, contribution conventions |
| [`developer/TROUBLESHOOTING.md`](developer/TROUBLESHOOTING.md) | Anyone stuck     | Common errors and how to fix them                    |

If you're brand new here: start with [SETUP.md](developer/SETUP.md). If you've
already run `make bootstrap` once, jump to [DEVELOPMENT.md](developer/DEVELOPMENT.md).
If you're driving an AI coding agent, read [`developer/AGENTS.md`](developer/AGENTS.md)
(and point the agent at [`../AGENTS.md`](../AGENTS.md) + [`../AI-CONTEXT.md`](../AI-CONTEXT.md)).
